Schema Entity-Relationship del Database

Modello dati PostgreSQL con quattro entità principali

Legenda

PK: Primary Key - Chiave primaria della tabella
FK: Foreign Key - Chiave esterna che referenzia altra tabella
UK: Unique Key - Campo con vincolo di unicità
JSONB: Tipo PostgreSQL per dati semi-strutturati con indicizzazione efficiente
Relazioni:
  • USERS → CHAT_SESSIONS: Un utente può avere più sessioni di chat (1:N)
  • USERS → ROOMS: Le prenotazioni sono embedded in JSONB con email utente (logica)
  • USERS → DOCUMENTS: Gli utenti consultano i documenti aziendali (1:N logica)
  • CHAT_SESSIONS → CHAT_MESSAGES: Una sessione contiene più messaggi (1:N)
